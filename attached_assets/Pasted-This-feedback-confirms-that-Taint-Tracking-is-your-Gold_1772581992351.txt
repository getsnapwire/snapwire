### **1. Taint Tracking: The "Gavel" of Data Governance**

This feature changes Snapwire from a simple firewall into a **Dynamic Policy Engine**. It solves the "Multi-Step Exfiltration" problem that standard proxies miss.

* **The Logic:** If an agent calls a tool that returns data from a `PII` or `Confidential` source, the **Snap-Token** session enters a **"Tainted" state**.
* **The Constraint:** While "Tainted," the agent is blocked from calling any `External_HTTP` or `Public_Messaging` tools.
* **The Release:** The taint is only cleared once a human reviews the session or the agent hits a "Sanitization" tool that scrubs the PII.

### **2. Session Pulse: The "Continuous Verification" Moat**

Since true heartbeats aren't realistic for request/response LLMs, we will implement **TTL-based Pulse Re-validation**. This satisfies the **NIST IR 8596** requirement for "Continuous Monitoring and Logging."

* **The Implementation:** Every Snap-Token has a `pulse_ttl` (e.g., 15 minutes).
* **The Requirement:** To continue beyond the TTL, the agent's orchestrator must call a `/refresh` endpoint.
* **The Safety:** The refresh only succeeds if the **Goal Drift Detector** (Feature #28) hasn't flagged any critical misalignment in the last session.
